Coding-Pages + Hexo 博客搭建

发表于 2019-06-27 | 更新于 2019-06-28 | 分类于 Blog ， Hexo
key ：Coding Pages Hexo 博客搭建 Jekyii Git Blog

参考：
https://hexo.io/zh-cn/docs/
https://www.jianshu.com/p/a19962485b4a

安装

1.自行注册Coding账号、安装node.js(npm)、Git Bash

2.新建文件夹blog 
    $ touch blog

3.安装hexo：$ npm install -g hexo  # 若npm太慢可自行安装cnpm

4.初始化 $ hexo init

5.本地打开查看是否运行成功
    $ hexo g    # hexo generate
    $ hexo s    # hexo server

http://localhost:4000 查看效果

切换主题

clone主题到themes文件夹下
    $ git clone git@github.com:iissnan/hexo-theme-next.git themes/next

1）安装好后，修改根目录下的_config.yml配置文件的theme属性为主题文件夹名 如：next
    注意：Hexo有两个config.yml文件，一个在根目录，一个在theme主题下。

2）清除缓存，应用主题
    $ hexo clean # 清除缓存 网页正常情况下可以忽略此条命令
    $ hexo g
    $ hexo s # 运行

修改配置

根目录下的_config.yml:
    deploy:
        type: git
        repository:
            coding: git@git.coding.net:nlbsn/blog.git
            #github: git@git.github.net:***/***.git     #还可以github也一起
        branch: master

在博客的source/目录下需要创建一个空白文件,至于原因，是因为 coding.net需要这个文件来作为以静态文件部署的标志。就是说看到这个Staticfile就知道按照静态文件来发布。
coding还需要在根目录下建一个.nojekyll空文件（coding page 配置hexo报错 the xx theme not be found 在根目录下建一个.nojekyll空文件就可以了，因为Coding默认支持Jekyll搭建网站）

    $ cd source/
    $ touch Staticfile 
    $ cd ../
    $ touch .nojekyll

codingPages自动部署

1.$ npm install hexo-deployer-git --save    # 安装这个，要不之后会报错error deployer not found:git

2.$ hexo clean      # 清除缓存
  $ hexo d -g       # hexo d || hexo deploy -global
